# 安全策略

感谢你帮助改进项目安全性。

## 报告范围

如果你发现以下问题，请优先私下报告，而不是直接公开提交：

- 凭据泄露
- 未授权访问
- 注入、远程执行、越权等高风险漏洞
- 依赖或部署流程中的敏感信息暴露

## 报告方式

请通过以下私密渠道联系维护者，并尽量提供：

- 创建 GitHub 私密安全通报（GitHub Security Advisory / Private vulnerability report）
- 或发送邮件到：`jing.empty@gmail.com`

- 问题描述
- 复现步骤
- 影响范围
- 修复建议（如果有）

在问题修复并确认发布前，请避免公开披露细节。


## 密钥管理建议

- 不要把 `.env`、私钥、Webhook、Token 提交到仓库
- 生产环境请使用环境变量、密钥管理服务或 GitHub Secrets
- 如果怀疑凭据已经泄露，请立即轮换相关密钥
