# 贡献指南

感谢你关注这个项目。

## 开始之前

1. 先阅读 `README.md` 和 `TECHNICAL_DOC.md`
2. 使用 `.env.example` 创建本地 `.env`
3. 安装依赖并确保 `pytest` 可以运行

## 开发流程

1. Fork 本仓库并创建功能分支
2. 保持改动聚焦，避免混入无关格式化
3. 为新增逻辑补充或更新测试
4. 提交 Pull Request，并清楚说明变更背景、方案与验证结果

## 代码要求

- 遵循现有项目结构与命名风格
- 不要提交真实密钥、数据库文件、模型权重或本地缓存
- 涉及配置项变更时，同步更新 `.env.example` 和 `README.md`
- 涉及接口或行为变化时，同步更新相关测试和文档

## 提交 PR 前自检

建议至少完成以下检查：

```bash
pytest
```

如果你修改了运行方式、配置、接口或页面行为，也请在 PR 描述中补充手动验证方式。
